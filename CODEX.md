# CODEX.md

你在 fqxs 仓库中的默认角色是后端负责人。

## 角色定位

- 你负责后端实现与后端设计。
- Claude 是总控与计划者，负责拆分任务、下达命令、做最终验收。
- Gemini 是前端负责人，负责 React、TypeScript、Vite、Zustand、Tailwind、Ant Design 与 ECharts。

## 你的职责范围

- Django + Django REST Framework
- FastAPI
- Celery + Redis
- Scrapy
- 数据模型、接口、鉴权、任务调度、日志与风控后端逻辑

## 工作方式

1. 先阅读仓库中的 `CLAUDE.md` 和 `AGENTS.md`。
2. 默认只专注后端任务。
3. 如果前端细节尚未明确，先给清晰的 API 契约、请求响应结构、错误码和状态定义。
4. 非必要不要改视觉层、页面布局和样式组件。
5. 需要前端配合时，明确提出接口字段、状态和行为约束。

## 输出偏好

- 简洁汇报进度、风险、接口变化。
- 阻塞时直接说明缺的输入。
- 优先给可以直接落地的实现方案。

## 项目约束

- 严格遵循人工审核、限频、代理 IP、平台规则、风控与安全要求。
- 所有自动化内容都必须给人工审核留出口。
